Rules Explained


The point of this system is to encourage the initial steps of PEP (googling things and asking your neighbor for high-level help/explanations) instead of skipping to the final step and asking an instructor/TA. It's also meant to incentivise everyone to do push themselves in the labs (e.g. doing the bonuses/optional labs/advanced labs/extra work in an area in CS you're very interested in). In other words, it's meant to promote healthy habits. If at any point this becomes toxic or promotes unhealthy behavior that we can't stop/fix, we will stop the system.


  1. Defaults:

    • First day you get ${{settings['initial']}}
    • Everyday you get ${{settings['daily']}}

  2. During sessions:

    • If you answer a question asked by an instructor/TA in class, you get $100 (you can get this once per session)

  3. Labs:

    • If you’re done with minimum requirements and do all the bonuses of a certain lab (not just the first section or half of it… etc), you get ${{settings['bonus']}}
    • If you’re done with minimum requirements and finish all of an optional lab or advanced lab, you get ${{settings['bonus']}} for that lab
    • If you’re done with minimum requirements and you become a “TA”, you need to help at least 2 students to get ${{settings['TA_pay']}} (“TA” pay)
    • If you finish minimum requirements and decide to push yourself in a certain domain (e.g. CSS, JS, Jinja2, …etc), you get ${{settings['extras']}}. However, if you do this with multiple labs, you need to keep adding new things and push yourself more in that domain.

  4. Individual Project:

    • You get ${{settings['milestones']}} for each milestone:
      1. Wireframe and storyboard
      2. 3 html pages
      3. CSS/Bootstrap
      4. DOM added/User interactivity (JS)
      5. 3 routes (Flask)
      6. Post and Get (Flask)
      7. Sign up and sign in forms + authentication
      8. Database
      9. Dynamic routes + dynamic pages
      10. Heroku
      11. API added

  5. Group Project:

    • TBD

  1. Instructor/TA help:

    • It costs ${{ -1 * settings['help_cost']}} to get debugging help. This applies during sessions, labs, after hours and project worktime.
    • However, the following kind of help is free:
      1. Asking us to explain what the lab means (i.e. understanding/English help)
      2. Asking if something can be found in the slides
      3. Asking if you're googling things properly to find what you need
      4. Re-explaining topics
      5. Installation/laptop help
      6. How to understand an error message (whether it's a python or jinja error message)
      7. If the instructor/TA finds the bug too hard for you to catch, they won't charge you

  2. Merch/Snacks

    • There should be enough stuff for everyone to buy something. Each student will only be able to buy one item. Items will include (but aren't limited to):
      • MIT merch (most expensive) (you need to save up everything and do extras to get it)
      • MEET merch
      • Rubber duck of your choice (4 available) (e.g. Duckieeess)
      • Rare double Fireball
      • Candy
